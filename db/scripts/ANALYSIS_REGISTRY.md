# 分析流程注册表

> 版本: v1.0 | 更新: 2026-03-27

## 分析报告模板结构

每份报告必须遵循以下章节顺序，承上启下：

```
一、数据概况
    ↓（为什么看这个）
二、核心发现
    ↓（基于数据发现的规律是什么）
三、Alpha 来源拆解
    ↓（归因到具体原因）
四、参数优化建议
    ↓（基于归因给出可操作建议）
五、结论与待办
```

## 分析步骤 ↔ 脚本 ↔ 数据来源 映射表

### 步骤 1: 数据加载

| 操作 | 调用函数 | 数据来源 | 字段 |
|------|---------|---------|------|
| 加载订单数据 | `load_order_data('k1', '2026-03-24')` | `db/files/订单数据/k1_订单数据_2026-03-24.xlsx` | 所有 54 列 |
| 加载 token 数据 | `load_token_data('k1', '2026-03-27')` | `db/files/token数据/k1_token数据_2026-03-27.xlsx` | 31 列 |
| 加载日报数据 | `load_daily_data('k1', '2026-03-26')` | `db/files/日报数据/k1_日报数据_2026-03-26.xlsx` | 9 列（含日期） |

### 步骤 2: 指标计算

| 指标名 | 调用函数 | 数据来源 | 计算公式 |
|--------|---------|---------|---------|
| gap | `calc_gap(df)` | 订单数据 | `gap = 单笔盈亏 - 带手续费理论利润` |
| slippage | `calc_slippage(df)` | 订单数据 | `slippage = gap - (-total_fee + bnb_singlePL)` |
| alpha | `calc_alpha(df)` | 订单数据 | `alpha = slippage + total_fee` |
| total_fee | 内嵌于 calc_slippage | 订单数据 | `total_fee = 捆绑费 + gas费u` |
| 费用率 | `calc_fee_rate(df)` | 订单数据 | `费用率 = 捆绑费 / 带手续费理论利润` |

### 步骤 3: 分组统计

| 统计项 | 调用函数 | 分组字段 | 数据来源 |
|--------|---------|---------|---------|
| 按方向统计 | `df.groupby('交易方向')` | 交易方向 | 订单数据 / 交易方向 |
| 按价差分组 | `df.groupby('pd_bin')` | pd_bin（价差区间） | 订单数据 / 价差 |
| 按规模分组 | `df.groupby('size_bin')` | size_bin（规模区间） | 订单数据 / 稳定币变化量u |
| 按状态分组 | `df.groupby('状态')` | 状态 | token数据 / 状态 |
| 按成功率分组 | `df.groupby('成功率_bin')` | 成功率区间 | token数据 / 成功率 |

### 步骤 4: 验算

| 验算项 | 调用函数 | 预期误差 | 数据来源 |
|--------|---------|---------|---------|
| 捆绑费验算 | `verify_bundle(df)` | < 0.01 | 订单数据 / 捆绑费, wbnb_bundle, WBNB价格 |
| 理论利润验算 | `verify_profit_virtual(df)` | < 0.01 | 订单数据 / 带手续费理论利润, 稳定币变化量u, 价差, 带手续费捆绑比例 |
| 单笔盈亏验算 | `verify_single_pl(df)` | 参考值 | 订单数据 / 单笔盈亏, token持仓价值, last_value, 稳定币变化量u, 捆绑费, gas费u |

## 报告数据来源标注规范

每个数据在报告中出现的格式：

```
数据名: {数值} {单位}
来源: {文件名} → {字段1} + {字段2} ... {计算公式}
```

示例：
```
总订单数: 53,755 笔
来源: k1_订单数据_2026-03-24.xlsx → COUNT(状态=1)

总费用: 7,568.11 U
来源: k1_订单数据_2026-03-24.xlsx → SUM(捆绑费) + SUM(gas费u)
公式: total_fee = Σ(捆绑费) + Σ(gas费u)

净 slippage: +8,172.90 U
来源: k1_订单数据_2026-03-24.xlsx → 公式 calc_slippage(df)
公式: slippage = Σ(gap) - Σ(-total_fee + bnb_singlePL)
```
