## 一、交易所类型

### CEX (中心化交易所)
- Binance, OKX, Coinbase 等
- 订单簿模式 (Order Book)
- 吃单 (Taker) vs 挂单 (Maker)
- 买一价 (Bid1)、卖一价 (Ask1)

### DEX (去中心化交易所)
- Uniswap, PancakeSwap, dYdX 等
- 自动做市商 (AMM)
- 流动性池 (Liquidity Pool)
- 滑点 (Slippage) 由 AMM 公式决定

---

## 二、AMM 工作原理

### 恒定乘积公式
```
x * y = k
```
其中 x 和 y 是两种资产的数量，k 是常数。

### 价格计算
```
价格 = y / x
```
每笔交易都会改变 x 和 y 的比例，导致价格滑动。

### 滑点 (Slippage)
- 预期价格 vs 实际成交价格的差异
- 交易规模越大，滑点越大
- AMM 中滑点由流动性池深度决定

## 三、MEV (最大可提取价值)

### 定义
矿工/验证者通过重新排序、插入或删除区块内的交易来提取的价值。

###常见MEV策略
| 策略 | 说明 |
|------|------|
| Frontrunning | 在目标交易前插入自己的交易 |
| Backrunning | 在目标交易后插入交易 |
| Sandwich Attack | 前后夹击目标交易 |

### MEV 提取方式
1. 套利机器人检测到价格差
2. 构造交易抢先执行
3. 从受害者交易中提取价值

## 四、Flashbots 和 Bundle

### Flashbots
- 以太坊的 MEV 研究组织
- 提供 Flashbots RPC，允许私密交易提交

### Bundle (捆绑交易)
- 多笔交易打包成一组
- 要么全部执行，要么都不执行
- 可避免 frontrunning

---

## 五、Gas 费用

### 定义
- 执行区块链操作需要的费用
- 以 ETH 或 BNB 支付
- Gas Price × Gas Used = 实际费用

### Gas 优化
- 减少计算步骤
- 批量操作减少固定开销
- 选择低拥塞时段

## 六、套利策略类型

### 1. 三角套利
```
USDT → ETH → USDT'
```
在同一交易所内循环交易，利用价格差。

### 2. CEX-DEX 套利
```
CEX 价格低 → DEX 价格高
在 CEX 买入，在 DEX 卖出
```

### 3. 跨交易所套利
```
Binance 价格低 → OKX 价格高
在 Binance 买入，在 OKX 卖出
```

## 七、你们的策略理解

### 策略流程
```
1. 采集 CEX 价格 (Bid1/Ask1)
2. 采集链上 DEX 价格
3. 计算价差 = |链上价格 - CEX价格|
4. 价差达到阈值 → 触发交易
5. 在 DEX 执行
6. 捆绑平台上链
```

### "买币/卖币"的含义
| 方向 | CEX | DEX | 目的 |
|------|-----|-----|------|
| 买币 | 按 Bid1 卖出 | - | - |
| 卖币 | 按 Ask1 买入 | - | - |

### 捆绑平台的作用
- 将多笔交易打包提交
- 原子性执行（要么全成功，要么全失败）
- 可能提供 Gas 优化
- 可能提供区块排序优势

## 八、滑点损耗的来源

### 1. AMM 滑点
- 交易规模大 → 价格滑动大
- 流动性不足时更明显

### 2. 执行延迟
- CEX 价格变动到 DEX 执行
- 市场快速波动时损耗大

### 3. MEV 提取
- 套利机器人检测到你的交易
- frontrunning 推高价格

### 4. 流动性不足
- 大额订单冲击成本
- 选择流动性好的交易对可减少

## 十二、触发机制

### 双边触发
用户策略：**CEX 和 DEX 两边都监控，哪边满足条件就触发哪边**

| 情况 | 条件 | 触发 |
|------|------|------|
| CEX 价格低 | CEX < DEX | 在 CEX 买入 |
| CEX 价格高 | CEX > DEX | 在 CEX 卖出 |
| DEX 价格低 | DEX < CEX | 在 DEX 买入 |
| DEX 价格高 | DEX > CEX | 在 DEX 卖出 |

### 稳定币交易量的作用
- `stable_amount_u` = 每次交易时稳定币的变化量
- 用于计算理论利润：`profit_virtual_fee ≈ |stable_amount_u| × price_diff`
- 正值 = 收到稳定币，负值 = 付出稳定币
