## 套利策略逻辑（核心）

### 策略触发机制
```
1. 采集价格
   ├─ 链上价格（DEX 价格）
   └─ 交易所价格（CEX 买一价 bid1 和卖一价 ask1）

2. 判断条件（CEX 数据仅用于触发）
   ├─ 情况A：链上价格 < CEX 买一价(bid1)
   │         → 计算 price_diff = |链上价格 - bid1|
   │         → 与阈值对比，达到则触发
   │
   └─ 情况B：链上价格 > CEX 卖一价(ask1)
             → 计算 price_diff = |链上价格 - ask1|
             → 与阈值对比，达到则触发

3. 执行交易（实际发生在链上 DEX）
```

## 核心公式（来源：MEMORY.md + alpha_归因分析.json）

```
single_PL = token_value - last_value + stable_amount_u - bundle - gasfee

gap = single_PL - profit_virtual

slippage = gap - (-total_fee + bnb_singlePL)
        = single_PL - profit_virtual - (-bundle - gasfee + bnb_singlePL)

alpha = slippage + total_fee
      = single_PL - profit_virtual + bnb_singlePL
```

> **新公式说明**：slippage 是核心 Alpha 来源，取代旧版 gap 公式。
> slippage 含义：扣除理论利润和所有费用后，价格朝有利方向移动的净贡献。

### token 状态含义
| 状态 | 含义 |
|------|------|
| **开启** | 允许该 token 进行买卖交易，策略正常运行 |
| **只卖** | 禁止买入，只允许卖出。适用于：行情不好或策略表现差，等待清仓 |
| **关闭** | 完全停止交易，该 token 不再参与策略 |
